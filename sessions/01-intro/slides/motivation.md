# Why This Meeting Series?

---

## The Landscape Is Moving Fast

- LLMs are rapidly changing how code gets written
- New tools and models appear every few weeks
- Hard to keep up individually

---

## Scientific Software Is Different

- Correctness matters — wrong results can invalidate research
- Performance-critical code (C++, Fortran, GPU kernels)
- Deep domain knowledge required (physics, math, numerics)
- Smaller user/developer communities than industry

---

## Our Goal

- Share experiences openly — what works, what doesn't
- Learn from each other's experiments
- Build collective knowledge at CCQ
- This is not advocacy — we want honest exchange
